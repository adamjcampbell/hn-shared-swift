# Android app

Gradle project that builds the Swift `AppCoreAndroid` target as a `.so`
library via the Swift Android SDK and consumes it through a JNI surface
generated by `swift-java jextract --mode=jni`.

## Modules

- `core-jni/` — Android library module. Runs `swift build` to produce
  `libAppCoreAndroid.so` + `libSwiftJava.so` for `aarch64-unknown-linux-android28`,
  stages those plus the Swift runtime `.so` files (and `libc++_shared.so`
  from the NDK) into `jniLibs/arm64-v8a/`, and exposes the
  jextract-generated Java sources from
  `AppCore/.build/plugins/outputs/.../JExtractSwiftPlugin/src/generated/java`.
  Also pulls in the `swiftkit-core-1.0-SNAPSHOT.jar` from the local
  `swift-java` checkout (provides `org.swift.swiftkit.core.SwiftLibraries`
  etc. that the generated Java references).
- `app/` — the actual Android application. `MainActivity`, `CityScreen`,
  `AppStateHolder` (a Compose state holder that implements the
  jextract-generated `SnapshotSink` Java interface), and `BridgePerfTest`
  instrumented tests under `androidTest/`.

## Prerequisites

| Component | Version |
|---|---|
| Swift toolchain | 6.3.1+ |
| Swift Android SDK | 6.3.1 release artefact bundle, configured with `ANDROID_NDK_HOME` |
| Android NDK | 27.x (we used 27.3.13750724) |
| Android SDK | platform 28+ + cmdline-tools |
| JDK | 21 (Android Studio's bundled JBR works) |
| `swift-java` CLI + Gradle artefacts | `swift build -c release --product swift-java` from a local checkout, plus `./gradlew :SwiftKitCore:build` once |
| `gradle` (one-time) | needed only to bootstrap `gradlew`; the wrapper is now committed |

## Local paths to update

`core-jni/build.gradle.kts` has two top-of-file constants you'll need to
adjust to match your install:

```kotlin
val swiftJavaToolDir = "/Users/adam/Developer/tools/swift-java"
val androidNdk       = "${System.getenv("HOME")}/Library/Android/sdk/ndk/27.3.13750724"
```

`local.properties` (gitignored) needs `sdk.dir` and `ndk.dir`. A starter
file:

```
sdk.dir=/Users/<you>/Library/Android/sdk
ndk.dir=/Users/<you>/Library/Android/sdk/ndk/27.3.13750724
```

## Build & run

```sh
# 1. Make sure the SDK's ndk-sysroot is wired to your local NDK, once:
ANDROID_NDK_HOME="$HOME/Library/Android/sdk/ndk/27.3.13750724" \
  bash "$HOME/Library/org.swift.swiftpm/swift-sdks/swift-6.3.1-RELEASE_android.artifactbundle/swift-android/scripts/setup-android-sdk.sh"

# 2. Build the APK. The first build runs jextract via the SwiftPM plugin
# and downloads Gradle 9.x for swift-java's SwiftKitCore subproject.
./gradlew :app:assembleDebug

# 3. Boot the AVD, install, launch.
emulator -avd Medium_Phone_API_36.1 -no-snapshot-load &
adb wait-for-device
adb install -r app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n com.example.appcore/.ui.MainActivity
```

## Tests

Instrumented tests measure JNI overhead, end-to-end round-trip latency, and
**enforce the cold-start regression** (initial snapshot must be delivered
synchronously from `appcoreCreate`, not from `Observations`'s mutation
emissions).

```sh
./gradlew :app:connectedDebugAndroidTest
adb logcat -d | grep BridgePerf
```

Representative numbers on `Medium_Phone_API_36.1` AVD (arm64-v8a, API 36):

| Test | n | median | p99 |
|---|---|---|---|
| Sync JNI call (`toggleFavorite`) | 200 | ~625 ns | ~7.5 µs |
| End-to-end round-trip | 100 | ~100 µs | ~370 µs |
| Snapshot JSON | — | 340 B / 357 B | — |

## Caveats

- **Sandbox.** SwiftPM's plugin sandbox blocks network access, but the
  `JExtractSwiftPlugin`'s Java-callback build phase needs to download
  Gradle for `:SwiftKitCore:build`. We pass `--disable-sandbox` to
  `swift build` (matching the upstream sample). The plugin's
  `GRADLE_USER_HOME` is also symlinked at `~/.gradle/wrapper` to reuse
  any existing Gradle distributions you've cached.
- **NDK version mismatch warnings.** AGP wants the exact NDK that the
  Swift Android SDK was built against; 27.3 is close enough — warnings
  are noisy but harmless.
- **Architectures.** Only `arm64-v8a` is built. Apple Silicon AVDs use
  arm64; physical Pixel/Galaxy devices are arm64-v8a. Add an `x86_64`
  swift build + ABI filter for x86_64 emulators on Intel Macs.
- **Strip warnings.** AGP can't strip Swift `.so` symbols; debug builds
  package them as-is, which inflates the APK to ~180 MB.
