# Android app

Gradle project that builds the Swift `AppCoreAndroid` target as `.so`
libraries via the Swift Android SDK and consumes them through a JNI surface
generated by `swift-java jextract --mode=jni`.

## Modules

- `core-jni/` — wraps the Swift build and the `jextract` step. The generated
  Java API class (`com.example.appcore.native.AppCoreNative`) lives under
  `core-jni/src/main/java/com/example/appcore/native/` and is **not** checked
  in (gitignored) — it is regenerated by the `jextractAppCoreAndroid` task.
- `app/` — the actual Android application. Hosts `MainActivity`,
  `CityScreen`, and the `AppStateHolder` Compose state holder.

## Prerequisites

| Component | Version |
|---|---|
| Swift toolchain | 6.3.1+ |
| Swift Android SDK | 6.3.1 release artefact bundle |
| Android NDK | 27d+ |
| JDK | 17+ |

Set the `swiftSdk` and `ndk` paths at the top of `core-jni/build.gradle.kts`
to point at your local install.

## Caveats

The Gradle task wiring is illustrative (see spec §6.1). The current setup
runs `swift build` for both `aarch64-unknown-linux-android28` and
`x86_64-unknown-linux-android28` and then `swift-java jextract`; you'll need
to add explicit task ordering (e.g. `jextractAppCoreAndroid` depends on the
two `buildSwift*` tasks, and the Android `assemble` lifecycle depends on
`jextractAppCoreAndroid`) to make `./gradlew assembleDebug` do everything in
one shot. See the
[`hello-swift-java`](https://github.com/swiftlang/swift-android-examples)
sample for a working reference.
